# 공연 검색 및 자동완성 연동 가이드 (FE)

| 버전 | 날짜 | 작성자 | 변경 내용 |
|------|------|--------|----------|
| 1.0  | 2026-03-07 | Gemini | 검색/자동완성 API 신규 가이드 작성 |

---

## 1. 개요
`smarter-store-api`의 공연 검색 시스템은 **Querydsl** 기반의 동적 필터링과 **17개 행정구역** 매핑을 지원합니다. 모든 검색 API는 인증 여부와 상관없이 `GET` 요청으로 호출 가능합니다.

---

## 2. 주요 API 명세

### 2.1 통합 검색 및 필터링
사용자가 입력한 키워드와 다중 선택된 필터를 기반으로 공연 목록을 반환합니다.

*   **Endpoint**: `GET /api/performances/search`
*   **Query Parameters**:
    *   `keyword` (String): 제목, 카테고리, 공연장 주소 통합 검색어
    *   `status` (List<Enum>): 판매 상태 필터 (`PerformanceSearchStatus`)
    *   `category` (List<String>): 장르(카테고리) 필터
    *   `region` (List<Enum>): 17개 행정구역 필터 (`Region`)
    *   `sort` (Enum): 정렬 방식 (`PerformanceSearchSort`) - 기본값: `CREATED_AT_DESC`
    *   `page` (Int): 페이지 번호 (0부터 시작)
    *   `size` (Int): 한 페이지당 아이템 수

*   **다중 값 전달 방식**: 
    표준 HTTP Query String 형식을 따릅니다. (Spring Boot에서 자동 리스트 변환 지원)
    `?status=ON_SALE&status=UPCOMING&category=뮤지컬&category=콘서트`

### 2.2 검색 자동완성
검색어 입력 시 실시간으로 최대 6개의 관련 공연을 추천합니다.

*   **Endpoint**: `GET /api/performances/search/autocomplete`
*   **Query Parameters**: `keyword` (검색어)
*   **특징**: 
    *   판매 종료된 공연은 제외하고 노출됩니다.
    *   응답에 포함된 `regionName`은 주소 매핑을 통한 17개 행정구역 명칭(예: "서울", "경남")입니다.

---

## 3. Enum 데이터 연동 (중요)

프론트엔드 UI(셀렉트 박스, 칩 필터 등) 구성을 위해 다음 Enum 정보를 사용하세요.

### 3.1 실시간 Enum 조회 API
전체 시스템 Enum 목록을 한 번에 가져와 캐싱하여 사용하세요.
*   **Endpoint**: `GET /api/enums`
*   **주요 키**:
    *   `regions`: 17개 행정구역 (서울, 인천, 경기, 강원 등)
    *   `performanceSearchStatuses`: 판매 상태 (판매 예정, 판매 중, 판매 종료)
    *   `performanceSearchSorts`: 정렬 옵션 (예매 많은 순, 종료 임박 순, 최근 등록 순)

### 3.2 Enum 상세 (Swagger 참조)
Swagger UI 상단의 **Schemas** 섹션에서 각 Enum의 상세 값($ref)을 확인할 수 있습니다.
*   `Region`: `SEOUL`, `INCHEON`, `GYEONGGI`, `GANGWON`, `CHUNGBUK`, `CHUNGNAM`, `DAEJEON`, `SEJONG`, `JEONBUK`, `JEONNAM`, `GWANGJU`, `GYEONGBUK`, `GYEONGNAM`, `DAEGU`, `ULSAN`, `BUSAN`, `JEJU`
*   `PerformanceSearchStatus`: `UPCOMING`, `ON_SALE`, `CLOSED`
*   `PerformanceSearchSort`: `BOOKING_COUNT`, `END_DATE_ASC`, `CREATED_AT_DESC`

---

## 4. 연동 팁 및 주의사항

1.  **인피니티 스크롤**: 응답 데이터의 `hasNextPage` 필드를 확인하여 다음 페이지 호출 여부를 결정하세요.
2.  **전체 개수 표시**: 응답 데이터의 `totalElements` 필드를 사용하여 상단에 "총 N개의 공연" 문구를 표시할 수 있습니다.
3.  **지역명 표시**: 서버 응답의 `regionName` 필드는 주소에서 추출된 "서울", "경기" 등 친숙한 명칭입니다. 필터링 전송 시에는 `Region` Enum 값(예: `SEOUL`)을 전송해야 합니다.
4.  **정렬**: "예매 많은 순(`BOOKING_COUNT`)"은 실제 결제 완료된 건수를 기준으로 서버에서 집계하여 제공합니다.
