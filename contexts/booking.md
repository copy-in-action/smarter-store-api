## Claude CLI 작업 Context

**Kotlin + Spring Boot**로 티켓 예매 시스템을 만들고 있어. **5분 예매 타이머** 기능을 구현해야 해.

### 핵심 요구사항

**타이머 관리:**
- 좌석 선택 페이지 진입 시 타이머 시작
- 좌석 선택부터 결제 완료까지 5분 카운트다운
- 5분 경과 시 예매 불가능하게 처리
- **서버 측에서 타이머 관리** (페이지 새로고침/브라우저 종료 대응)
- 남은 시간을 API로 프론트엔드에 전달

**동시성 제어 (구현 예정):**
- 타이머 기간 동안 좌석 점유 관리
- 결제 완료 검증
- 적절한 락 메커니즘으로 중복 예매 방지

### 현재 프로젝트 상태
- **백엔드**: Kotlin + Spring Boot (smarter-store-api)
- **데이터베이스**: PostgreSQL
- **캐시**: Redis 미사용 (필요하면 추가 가능)
- **프로젝트 단계**: 설계 중

### 기술적 제약사항
- Redis 현재 미구성 (추가 가능)
- 프론트엔드 개발자가 별도로 있어서 나는 API만 제공하면 됨
- 서버 시간 기준으로 페이지 새로고침/재접속 시에도 타이머 유지 필요

### 예상 동작 흐름
1. 사용자가 좌석 선택 페이지 진입 → 타이머 시작 (5분)
2. 좌석 선택 후 결제 진행
3. 5분 경과 시 → 프론트에서 모달 표시, 예매 불가 처리
4. 시간 내 결제 완료 → 예매 성공
5. 페이지 새로고침 시 → 서버에서 남은 시간 기준으로 계속 진행

### 도움이 필요한 부분
1. 타이머 관리 API 엔드포인트 설계 (시작, 남은 시간 조회, 취소)
2. 활성 예약 정보를 저장할 적절한 자료구조 선택 (인메모리 vs DB vs Redis)
3. 세션/예약 ID 기반 서버 측 타이머 추적 구현
4. 좌석 점유를 위한 동시성 제어 전략 설계
5. 엣지 케이스 처리 (결제 진행 중 타이머 만료 등)
